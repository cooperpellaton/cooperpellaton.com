  • PrintShare

    A four month medical adventure.

    PrintShare was a noble attempt to revolutionize document sharing in the health setting. For the uninitiated, documents are currently faxed from medical institution to medical institution and can take up to 3 weeks to process. Aaron Aabajin and myself designed this end to end solution which would enable a health service center to share a medical record direct from an Electronic Health Record (EHR) system to anywhere else without printing. If the reciever so happened to be using PrintShare the Electronic Medical Record would also be immediately inserted into their EHR.

    We were unable to raise further funding for this project and development stopped in May 2016. Our secure fileupload API is available here & you can read about the entire experience here.

  • PGE - Pretty Good Email

    This project is in active development.

    What is PGE you say to yourself? PGE means "Pretty Good Email," which is an email encryptor based on the Pretty Good Protection specification. The core selling point of PGE is that all encryption happens locally, on the client computer before any data is transmitted. Thus, one can rest assured that even their most secure of secrets will not fall into the wrong hands during transmission as the only time the unencrypted file will be seen is on the client computer at the time of the emails inception, and at the time of opening by the intended recipient.

    Using OpenPGP.js and the Keybase API as well as KBPGP.js we check a users email against existing accounts to figure out their public keys and then use this information to encrypt her email. This is currently written to work with FastMail and expressly being tested as a Greasemonkey/Tampermonkey script. It is the intention of this project to bundle it as a Chrome extension, which given proper care in our XMLHttpRequest header will be able to parse data coming from any email provider and thus render the same surface.

  • SniffleGiggle

    This is a product of HampHack '16.

    SniffleGiggle you say questioningly to yourself...what is it? My friend, Thomas Lam, told me about a scence in Parks and Rec referenced in our readme where a character, Tommy, shares his talking tissues with colleagues. What we've built here is just that, a talking, sassy tissue box.

    Built using an Intel Edison, a plethora of sensors and some node, this was a quick hardware hack that we cobbled together. We turned the Edison into a web server running Node with Express and Socket.io to send events to the client and tell it to play a random pre-recorded sound. This was built with Node.js, Socket.io, Johnny-Five and an Intel Edison.

  • Hotline-Bing

    Won best use of Priceline API at YHACK-2015. Won 2nd best use of HP Haven-On-Demand API.

    A product of YHACK-2015, Hotline-Bing is an SMS driven web-search client. Search the internet and used internet based services when you have no internet! Based on initial user input as stored in note the query is either handed off to Haven-On-Demand where text analysis is done and the proper dates and locations are returned, or given directly to Bing.

    If the user indicates that she wants to look for a hotel on Priceline than the API is queried with the users input and the top hotels are returned. The user gets to see the top hotel listings in the selected area with the option to learn more about any of them. If they user indicates that they would like to pay then a link to the Braintree payments system is sent to them where they can then pay for the hotel room at that time.
